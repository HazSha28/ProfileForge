ProfileForge Documentation

ProfileForge transforms scattered candidate data from multiple sources into a single validated, scored canonical profile — with full provenance on every field. It supports both single-candidate and bulk batch processing.

📥
Input

Recruiter CSV + Resume PDF + Platform links. Bulk: CSV + ZIP of resumes.

⚙️
Pipeline

Parse → Normalise → Merge → Confidence → Validate → Project

📤
Output

Canonical JSON with sources + confidence per field. One file per candidate.

Quick Start — Single Candidate

  1. 1
    Go to New Profile

    Click New Profile in the sidebar or dashboard.

  2. 2
    Upload a Recruiter CSV

    One row per candidate. Column names are flexible — see the CSV section below.

  3. 3
    Upload a Resume PDF

    Text-based PDFs only (not scanned images). AI extraction runs if a Gemini API key is configured.

  4. 4
    Click Generate Profile

    Pipeline runs in seconds. Review the profile, then download the JSON.

Resume File (PDF or DOCX)

ProfileForge accepts both PDF and DOCX (Word) resumes. It uses AI (Groq/Gemini) when configured, or falls back to regex extraction.

Scanned PDFs are not supported — they contain no extractable text. Use text-based PDFs or DOCX files only.

Bulk Processing

How Bulk Processing Works

  1. 1
    Upload Recruiter CSV

    One row per candidate. Supports Name of the Student, Registration Number, GitHub/LinkedIn URLs, etc.

  2. 2
    Upload Resume ZIP

    A ZIP archive containing all PDF resumes. Filenames should include the candidate's name, email, phone, or registration number.

  3. 3
    Match Resumes to CSV Rows

    The matching engine runs 6 strategies in priority order (see below). Processing never stops — unmatched records are flagged as warnings.

  4. 4
    Full Pipeline per Candidate

    Each matched pair runs Parse → Normalise → Merge → Confidence → Validate → Project.

  5. 5
    Download Results

    One candidate_NNN.json per candidate + bulk_summary.json with job stats. View in Bulk History.

Resume Matching Strategy

Matching uses a deterministic 6-tier priority chain. The highest-priority match wins. If multiple resumes match, the highest-score one is chosen.

1
Email Email found in resume filename or PDF content matches CSV email
2
Phone Phone extracted from resume matches CSV phone (last-10-digit comparison, E.164-aware)
3
Reg Number Registration/roll number (e.g. 711523BCB041) found in resume filename
4
GitHub User GitHub username from CSV URL found in resume filename
5
Exact Name Normalised name from filename matches CSV name (strips initials, accents, punctuation)
6
Fuzzy Name RapidFuzz token_set_ratio ≥ 90% — handles middle names, initials, word order
Name normalisation: "AYISHATHUL HAZEENA S" → "ayishathul hazeena" (strips single-char initials, accents, dots, commas, apostrophes)

Merge Policy

Deterministic — no randomness. Source priority: Resume > CSV.

Scalar (name, headline, years)Highest-priority source with a non-null value wins
Lists (emails, phones, skills)Deduplicated union — Resume values first
Nested (location, links)Each sub-field merged independently with scalar rule
Missing everywherenull

Email Notifications

ProfileForge can send a summary email when a bulk processing job completes — showing total processed, average confidence, failed, and resume-missing counts.

Setup (Gmail recommended):

  1. 1
    Enable 2FA

    Go to your Google account → Security → 2-Step Verification → enable it.

  2. 2
    Generate an App Password

    myaccount.google.com → Security → App Passwords → select Mail → Generate. Copy the 16-character password.

  3. 3
    Add to .env

    Set EMAIL_USER, EMAIL_PASSWORD, and EMAIL_HOST=smtp.gmail.com.

EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your@gmail.com
EMAIL_PASSWORD=xxxx xxxx xxxx xxxx
EMAIL_FROM_NAME=ProfileForge
If EMAIL_HOST / EMAIL_USER / EMAIL_PASSWORD are not set, email is silently skipped — bulk processing always completes regardless.

FAQ

How do I enable dark mode?

Click the moon icon (🌙) in the topbar. It toggles between light and dark mode and remembers your preference across page loads via localStorage.

Why isn't the email notification being sent after bulk processing?

Email is only sent if EMAIL_HOST, EMAIL_USER, and EMAIL_PASSWORD are all set in your .env file. If any of them is missing, email is silently skipped and bulk processing still completes normally.

My deployed app on Render is slow / times out.

The free tier spins down after 15 minutes of inactivity and takes 30–60 seconds to cold start. Add a free UptimeRobot monitor pinging /api/health every 5 minutes to keep it warm. Alternatively, upgrade to Render's paid tier ($7/month) for always-on instances.

My CSV has "Name of the Student" — will it work?

Yes. The bulk processor uses partial column-name matching, so Name of the Student, Candidate Name, full_name, and name all map to the name field automatically.

All my candidates show "No Resume" — why?

The matcher reads resume filenames first. If filenames don't contain the candidate's name, email, phone, or registration number, it falls back to reading PDF content. Name your files like PRAKASH B-711523BCB041.pdf for the best results.

Skills are not being extracted from the resume.

Ensure your resume has a clearly labelled section — Skills, Technical Skills, Soft Skills, Extracurricular, etc. If using AI extraction (Gemini key configured), skills are extracted from all sections automatically.

Phone shows as empty even though it's in the PDF.

Phone normalisation requires a country hint. If the CSV doesn't have a country column, add "IN" or "US" to the country field, or include the country code in the phone number itself (e.g. +91 9876543210).

What encoding should my CSV be saved in?

Any encoding works. ProfileForge automatically tries UTF-8, UTF-8-BOM, Windows-1252, and Latin-1 — files exported from Excel are handled correctly.

Can I add my own skill aliases?

Yes. Open src/normalizers/skills.py and add entries to the SKILL_ALIASES dictionary. Key is any lowercase alias, value is the canonical display name.

How do I deploy ProfileForge?

The repo includes render.yaml and Procfile. Go to render.com, connect your GitHub repo, add your env vars, and click Deploy. Update your Google/GitHub OAuth redirect URIs to the new domain after deployment.
