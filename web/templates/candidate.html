What happens next

  1. CSV is parsed — recruiter data extracted with flexible column mapping
  2. PDF is parsed — AI or regex extracts name, contacts, skills, experience
  3. Data is normalised — phones → E.164, country → ISO Alpha-2, skills → canonical
  4. Records are merged — Resume wins over CSV; lists are unioned
  5. Confidence assigned — each field scored by how many sources agree
  6. Schema validated — required fields, email format, phone format checked
  7. Profile generated — canonical JSON with provenance on every field
